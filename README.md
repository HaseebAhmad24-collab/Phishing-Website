A specialized frontend demonstration project designed for **cybersecurity education** and **social engineering awareness**. This repository contains a high-fidelity recreation of common web interfaces used in phishing simulations to help developers and security professionals identify deceptive UI/UX patterns.

## 🚀 Features

- **High-Fidelity UI:** Realistic landing pages and login forms designed to mimic modern web services.
- **Multi-Step Flow:** Includes a complete user journey from landing (`index.html`) to data entry (`order.html`) and confirmation (`result.html`).
- **Integrated Logic:** Demonstrates how JavaScript can be embedded within HTML to handle user interactions and data flow.
- **Asset Management:** Optimized use of images (`.jpg`, `.webp`) to create a convincing visual experience.
- **Lightweight:** Pure HTML/JS/CSS implementation with no heavy dependencies.

## 📂 Project Structure

- `index.html`: The primary landing page/entry point for the simulation.
- `order.html`: A simulated form for capturing user input or "order" details.
- `result.html`: A redirection or success page displayed after user interaction.
- `js_with_html.html`: Educational examples of scripting within a deceptive environment.
- `*.jpg / *.webp`: Visual assets used to enhance the realism of the frontend.

## 🛠️ Installation Guide

Since this is a static frontend project, no complex installation or server-side setup is required to view the UI.

1. **Clone the repository:**
   ```bash
   git clone https://github.com/HaseebAhmad24-collab/Phishing-Website.git
   ```

2. **Navigate to the directory:**
   ```bash
   cd Phishing-Website
   ```

3. **Open the project:**
   Simply double-click `index.html` to open it in your default web browser, or use a "Live Server" extension in VS Code.

## 💡 Usage Examples

### Security Training
Use these templates in a controlled environment to teach employees or students how to inspect URLs, verify SSL certificates, and look for "red flags" in UI elements before entering credentials.

### UI/UX Research
Study how visual cues (like `login image.jpg` and `map.webp`) are used to build trust with a user in a digital environment.
